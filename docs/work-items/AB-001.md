## Background

Data- och analysportalen är i ett tidigt mockup-/prototypstadium. Projektet hade ingen strukturerad, spårbar arbetsprocess för att skapa, följa upp, dokumentera och validera arbetsuppgifter, och docs/-katalogen innehöll 15 detaljerade referensdokument (00-14) som är för omfattande för att läsas i sin helhet inför varje mindre AI-assisterad ändring.

Verktyget Worksmith (tools/worksmith/) tillhandahåller ett självständigt kit (core/, scaffold/, examples/) avsett att kopieras in i ett projekt för att ge deterministisk work-item-hantering (skapa, validera, övergå livscykelstatus, slutföra) samt en mall för lättviktig, operativ projektdokumentation.

## Goal

Införa Worksmith som projektets arbetssätt för work items, och samtidigt skapa en tydlig tvånivåstrategi för dokumentation: befintliga docs/00-14 kvarstår oförändrade som tung referensdokumentation, medan ett nytt lättviktigt operativt lager (AGENTS.md och docs/project/*) styr vad en AI-agent eller utvecklare behöver läsa för en given uppgift.

## Scope

- Kopiera tools/worksmith/core/ verbatim till scripts/ (projektadministrations-CLI:t).
- Skapa en minimal root package.json med Worksmith-scripts (project, project:test, m.fl.) och installera tsx/typescript som devDependencies, utan att röra frontend/package.json.
- Fylla i tools/worksmith/scaffold/-mallarna som aktiva projektdokument: AGENTS.md (repo-rot) samt docs/project/PROJECT_RULES.md, PROJECT_WORKFLOW.md, WORK_ITEM_SCHEMA.md, DOCUMENT_INDEX.md, PROJECT_STATUS.md, DECISIONS.md, PROJECT_CONTEXT.md och docs/development/PROJECT_ADMINISTRATION.md.
- Skapa .worksmith/topics.json (projektets ämnesvokabulär) och .worksmith.json (svensk CLI-presentation, projektnamn).
- Formulera och dokumentera en lässtrategi i docs/project/DOCUMENT_INDEX.md som säger vilka dokument som ska läsas för vilken typ av ändring, utan att förkorta eller ersätta docs/00-14.
- Köra npm run project -- init/validate samt npm run project:test och verifiera resultatet.
- Skapa detta första work item som en praktisk demonstration av arbetssättet.
- Genomföra en enkel säkerhetsgenomgång av alla nya filer (inga interna URL:er, secrets, certifikat, riktiga AD-grupper eller personuppgifter).

## Requirements

- scripts/ ska vara en byte-för-byte-identisk kopia av tools/worksmith/core/ (verifierat mot MANIFEST.json:s sha256-hashar) — ingen intern logik ändras utan dokumenterad anledning.
- Aktiv Worksmith-dokumentation ska ligga under docs/project/ och docs/development/, aldrig permanent under tools/worksmith/.
- Inga scaffold-placeholders (<PLACEHOLDER>, {{PLACEHOLDER}}) får finnas kvar i de aktiva dokumenten.
- Befintlig referensdokumentation (docs/00_Projektprinciper.md till docs/14_Rekommenderade_extensions.md) ska bevaras oförändrad och förbli den auktoritativa detaljkällan.
- Alla nya konfigurationsfiler (.worksmith/topics.json, .worksmith.json, package.json) ska vara fria från interna URL:er, secrets, connection strings och andra känsliga värden.
- Ämnes-id:n i .worksmith/topics.json får inte krocka (som citerad JSON-sträng) med ord som redan förekommer i tools/worksmith/-kitets egna scaffold-/exempelfiler.

## Verification

- npm run project -- init --dry-run följt av npm run project -- init skapade docs/work-items/ och docs/WORK_QUEUE.md utan fel.
- npm run project -- validate returnerade 0 fel, 0 varningar.
- npm run project:test kördes: 84 av 86 tester passerade. De två återstående testerna kräver scripts/build-worksmith-kit.ts respektive scripts/project.ts, vilka avsiktligt inte ingår i det distribuerade kitet (kitets egna interna underhållsverktyg) — dokumenterat i docs/development/PROJECT_ADMINISTRATION.md.
- npm run project -- config --output terminal visade korrekt projektnamn (Data- och analysportalen) och svenskt språk.
- En Python-baserad sha256-kontroll bekräftade att samtliga 32 kopierade filer i scripts/ matchar tools/worksmith/MANIFEST.json exakt.
- En textsökning bekräftade att inga interna URL:er, lösenord, tokens, connection strings eller certifikat förekommer i de nytillkomna filerna.

## Acceptance Criteria

- Worksmith core är kopierad till scripts/.
- Worksmith scaffold är kopierad till aktiv projektdokumentation (AGENTS.md, docs/project/*, docs/development/PROJECT_ADMINISTRATION.md).
- Placeholders i aktiva Worksmith-dokument är ersatta med projektspecifikt innehåll.
- package.json (repo-rot) innehåller nödvändiga Worksmith-scripts utan att påverka frontend/package.json.
- Worksmith init och validate har körts och passerat.
- Projektets befintliga dokumentation (docs/00-14) är indexerad i docs/project/DOCUMENT_INDEX.md.
- Strategi för tung referensdokumentation kontra lätt operativ dokumentation är bedömd och dokumenterad (docs/project/DECISIONS.md, DEC-002).
- Inga interna URL:er, secrets eller känsliga värden har införts.

## Out of Scope

- Att bygga en backend (.NET Web API) eller applikationsdatabas (PostgreSQL) — detta är ett separat, framtida arbete enligt docs/04_Systemarkitektur.md.
- Att korta ner, slå ihop eller ta bort innehåll i docs/00_Projektprinciper.md till docs/14_Rekommenderade_extensions.md.
- Att ändra scripts/project-administration/configuration.ts:s värden (prefix, sökvägar, applicationName) — dessa hålls oförändrade eftersom det kopierade testsviten verifierar dem exakt (se docs/project/DECISIONS.md, DEC-001).
- Att slutföra (transition till done) detta work item utan explicit godkännande från projektägaren.
